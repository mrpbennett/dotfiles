# LazyVim Keybindings for Zed

A comprehensive, drop-in replacement keymap that brings LazyVim's ergonomic keybindings to the Zed editor. Perfect for developers who love LazyVim's workflow but want to use Zed's modern, high-performance architecture.

## Quick Start

```bash
# Run the installation script
./install.sh
```

Or manually:

```bash
# Backup existing keymap (if any)
cp ~/.config/zed/keymap.json ~/.config/zed/keymap.json.backup

# Copy the keymap
cp keymap.json ~/.config/zed/keymap.json

# Enable Vim mode in Zed settings
# Add to ~/.config/zed/settings.json:
{
  "vim_mode": true,
  "vim": {
    "use_system_clipboard": "always",
    "use_smartcase_find": true,
    "toggle_relative_line_numbers": true
  }
}
```

## What's Included

This keymap replicates **100+ LazyVim keybindings** including:

- ✅ **Window Management** - `Ctrl-h/j/k/l`, splits, resizing
- ✅ **Buffer Navigation** - `Shift-H/L`, `[b`/`]b`, buffer switching
- ✅ **Line Movement** - `Alt-j/k` to move lines up/down
- ✅ **LSP Operations** - Go to definition, references, code actions
- ✅ **Git Integration** - Git panel, blame, browse
- ✅ **Search & Find** - File finder, project search, symbols
- ✅ **Diagnostics** - Navigate errors, show diagnostics panel
- ✅ **Terminal** - Toggle terminal with `Ctrl-/`
- ✅ **Leader Key** - `Space` as leader with consistent LazyVim mappings
- ✅ **Vim Enhancements** - `jk` to escape, better search, visual line movement

## Key Features

### 🎯 Leader Key Mappings
All LazyVim leader key combinations are preserved using `Space`:
- `<leader>ff` - Find files
- `<leader>fg` - Project search (grep)
- `<leader>bd` - Close buffer
- `<leader>ca` - Code actions
- And 50+ more...

### ⚡ Muscle Memory Friendly
Direct 1:1 mappings where possible:
- Window navigation: `Ctrl-h/j/k/l`
- Buffer switching: `Shift-H/L`
- Splits: `<leader>-` and `<leader>|`
- Git: `<leader>gg`

### 🔧 Vim Mode Integration
Seamless Vim mode support:
- `jk` to escape insert mode
- Visual mode line movement maintains selection
- `Ctrl-w` prefix for window commands
- Go to definition/references: `gd`, `gr`

## What's Different from LazyVim?

While this keymap replicates most LazyVim bindings, some features are unavailable due to Zed's architecture:

- Window zoom toggle (no zoom feature yet)
- Zen mode (not implemented)
- Treesitter toggle (different syntax system)
- Snacks.lua integrations (different plugin system)
- Some toggle options (controlled via settings instead)

See [LAZYVIM_KEYBINDS.md](./LAZYVIM_KEYBINDS.md) for a complete list.

## Customization

The keymap is organized with clear comments. To customize:

1. Open `~/.config/zed/keymap.json`
2. Find the section you want to modify
3. Add/remove/modify bindings
4. Save and restart Zed

Example - Add a custom binding:
```json
{
  "context": "VimControl && !menu",
  "bindings": {
    "space m m": "markdown::OpenPreview"
  }
}
```

## Troubleshooting

### Keybindings not working
- Ensure Vim mode is enabled in Zed settings
- Restart Zed after installing the keymap
- Check for JSON syntax errors

### Leader key not responding
- Make sure you're in Normal mode (press `Esc` first)
- Wait ~1 second after pressing `Space` for the sequence
- Verify `vim_mode: true` in settings

### Conflicts with defaults
- Your keymap overrides Zed defaults
- To disable a default binding, set it to `null`
